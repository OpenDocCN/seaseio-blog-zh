// first talk

用弹性堆栈丰富邮政地址

大多数时候，我们的客户或用户的邮政地址在我们的信息系统中不是很好地格式化或定义的。举例来说，如果你是一个呼叫中心的员工，想通过客户的地址找到客户，这可能会成为一场噩梦。想象一下，销售服务如何轻松地在地图上标出客户的位置，以及他们可以在哪里开设新店……
让我们举一个简单的例子:

{
  "name": "Joe Smith",
  "address": {
    "number": "23",
    "street_name": "r verdiere",
    "city": "rochelle",
    "country": "France"
  }
}

或者相反。我有坐标，但我不知道对应的邮政地址是什么:

{
  "name": "Joe Smith",
  "location": {
    "lat": 46.15735,
    "lon": -1.1551
  }
}

在这个现场编码会议中，我将向您展示如何使用Elastic stack解决所有这些问题，重点是Logstash和Elasticsearch。
